# 数据说明

本目录和 GitHub 公开同步。表内公司与联系方式为项目数据，不做脱敏。API Key 不属于项目数据，不写入此目录。

## 数据分层

- `raw/upkuajing/2026-07-17/`：跨境魔方搜索任务的原始 JSONL 与任务元数据，不手工修改。
- 同一目录还保存公司联系方式、人物补救、邮箱验证和电话验证的完整响应；处理脚本默认复用这些文件，不重复付费。
- `raw/web/2026-07-17/`：官网、政府文件和权威目录的核验证据。
- `processed/company-master.json`：Excel 的结构化输入，一行对应一个 `companyId`。
- `processed/initial-six-contact-enriched-leads.*`：已取回联系方式的首批 6 家公司，是历史处理快照，不是最终主表。
- 最终、唯一权威客户总表位于 `../deliverables/tape-masking-film-customer-master.xlsx`。

## 完整性规则

1. 每次付费调用的返回记录都保留在 `raw/`。
2. 每家公司在 Excel 只占一行，以 `companyId` 为主键。
3. 不丢弃 API 字段；主表保留业务可读字段，完整原始值仍可从 JSONL 追溯。
4. 邮箱、电话、网站、社媒保留有效性状态和来源。
5. 未补齐邮箱的公司不删除；电话或官网等有效渠道仍保留在同一行。
6. 核心业务指标以公司为单位：50 家拥有至少一种有效联系方式，39 家拥有有效邮箱，41 家拥有有效电话；覆盖结构为 30 家两者都有、9 家只有邮箱、11 家只有电话、0 家两者都没有。
7. 邮箱技术指标单独表达：46 个跨公司去重后的有效邮箱，49 条公司—有效邮箱关联记录。
8. 本批 OpenAPI 累计费用为 ¥105.10；任何新增付费调用都必须在 ¥200 累计上限内。
9. 结构化主数据和原始响应保留 API 原文；Excel 展示层将品类、搜索词、国家及常用产品术语本地化为中文，无法可靠翻译的型号和缩写保留原样。
