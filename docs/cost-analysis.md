# 有效联系方式成本与流程优化

核算日期：2026-07-17。目标口径不妥协：只有邮箱状态 `1` 才计入“有效邮箱”；官网存在但验证状态为 `3` 或 `0` 的邮箱不计入主指标。

## 最终结果

| 指标 | 结果 |
|---|---:|

| 每家有效联系方式成本 | ¥2.33 |

这三个邮箱口径不能混用：

- `26 家公司`：26 家公司各自至少有一个状态为 `1` 的邮箱。这是衡量获客覆盖的核心指标，`¥81.60 ÷ 26 = ¥3.14/家`。
- `28 个去重邮箱`：把全部有效邮箱文本放在一起跨公司去重后，共有 28 个独立地址，`¥81.60 ÷ 28 = ¥2.91/个`。
- `31 条公司—邮箱关联`：同一个邮箱出现在两家公司行中会计算两次，因此是 31 条关联记录，不等于 31 个独立邮箱。`¥2.63/条`仅用于数据审计，不作为核心获客成本。

本批重复关联来自 `support.in@vivo.com`、`vcare@vivo.com`、`marketing@ajitindustries.com`，三者各在两家公司行中出现一次重复。

## 下一版流程

1. 搜索结果入库后，先按标准化公司名、官网域名、国家做法人实体去重；同实体直接复用已验证联系方式。
2. 公司联系方式只购买一次；原始响应以公司ID做幂等缓存。
3. API、官网、政府文件和人物邮箱进入同一增量验证队列，不再按来源分开漏验。
4. 无有效邮箱时先免费查官网；仍没有再逐公司搜索人物。只购买公司精确匹配且岗位相关的第一位人员。
5. 状态 `3` 多为验证服务器被反垃圾系统拦截，状态 `0` 表示接口已调用但没有完成检测；二者都不重复盲测，改走新人物或真实投递闭环。
6. 正式群发前先小批量发送，记录硬退信、软退信与回复；硬退信地址永久进入抑制名单。验证状态 `1` 是筛选门槛，不是送达保证。

按本批数据估算，若前置实体去重、取消一次混合人物搜索，并避免对明显错误实体逐公司搜索，可至少节省约 ¥8.50；有效邮箱结果不应因此减少。
