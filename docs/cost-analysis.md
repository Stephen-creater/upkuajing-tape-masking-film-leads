# 有效联系方式成本与流程优化

核算日期：2026-07-17。目标口径不妥协：只有邮箱状态 `1` 才计入“有效邮箱”；官网存在但验证状态为 `3` 或 `0` 的邮箱不计入主指标。

## 公司级核心业务指标

| 指标 | 公司数 | 获客成本 |
|---|---:|---:|
| 拥有至少一种有效联系方式的公司 | 50 | ¥2.10/家 |
| 拥有有效邮箱的公司 | 39 | ¥2.69/家 |
| 拥有有效电话的公司 | 41 | ¥2.56/家 |

三个指标都以公司为单位：同一家公司无论有多少个有效邮箱或电话，都只计算一次。三项成本均以累计 API 费用 ¥105.10 为分子，分别除以对应的公司数，不能相加。

## 公司覆盖结构

| 覆盖情况 | 公司数 |
|---|---:|
| 邮箱和电话都有 | 30 |
| 只有有效邮箱 | 9 |
| 只有有效电话 | 11 |
| 两者都没有 | 0 |

因此，前述 11 家没有有效邮箱的公司全部拥有有效电话，50 家公司没有联系方式空白。

## 邮箱技术审计

- `39 家公司`：39 家公司各自至少有一个状态为 `1` 的邮箱。这是公司级业务指标，`¥105.10 ÷ 39 = ¥2.69/家`。
- `46 个去重邮箱`：把全部有效邮箱文本放在一起跨公司去重后，共有 46 个独立地址；这是技术审计数量，不计算公司级获客成本。
- `49 条公司—邮箱关联`：同一个邮箱出现在两家公司行中会计算两次，因此是 49 条关联记录，不等于 49 个独立邮箱。该数字仅用于数据审计，不作为核心业务指标。

本批重复关联来自 `support.in@vivo.com`、`vcare@vivo.com`、`marketing@ajitindustries.com`，三者各在两家公司行中出现一次重复。

OpenAPI 累计费用为 ¥105.10，预算剩余 ¥94.90。

## 费用构成

| 环节 | 金额 |
|---|---:|
| 海关客户搜索 | ¥12.00 |
| 海关公司联系方式 | ¥61.00 |
| 人物搜索 | ¥19.50 |
| 人物联系方式 | ¥3.00 |
| 邮箱验证 | ¥3.00 |
| 电话验证 | ¥6.60 |
| 合计 | ¥105.10 |

人物补救阶段先人工核对公司和职位，只购买6人的联系方式，实际恢复了 Samsung、Haier、Super Sales 三家公司的有效邮箱。人物搜索共花费 ¥19.50，是本轮最需要优化的环节：名称宽泛的公司容易返回大量错误实体。

## 下一版流程

1. 搜索结果入库后，先按标准化公司名、官网域名、国家做法人实体去重；同实体直接复用已验证联系方式。
2. 公司联系方式只购买一次；原始响应以公司ID做幂等缓存。
3. API、官网、政府文件和人物邮箱进入同一增量验证队列，不再按来源分开漏验。
4. 无有效邮箱时先免费查官网；仍没有再逐公司搜索人物。只购买公司精确匹配且岗位相关的第一位人员。
5. 状态 `3` 多为验证服务器被反垃圾系统拦截，状态 `0` 表示接口已调用但没有完成检测；二者都不重复盲测，改走新人物或真实投递闭环。
6. 正式群发前先小批量发送，记录硬退信、软退信与回复；硬退信地址永久进入抑制名单。验证状态 `1` 是筛选门槛，不是送达保证。

本轮从 35 家扩充到 50 家时，没有再次调用高成本人物搜索：3 次增量公司搜索花费 ¥4.50，19 家公司联系方式花费 ¥19.00，新增 15 家有效联系方式公司，边际成本为 ¥1.57/家。该结果支持“先批量公司联系方式、状态1即停止、人物搜索仅作最后补救”的优化流程。
