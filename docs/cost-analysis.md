# 有效联系方式成本与流程优化

核算日期：2026-07-17。目标口径不妥协：只有邮箱状态 `1` 才计入“有效邮箱”；官网存在但验证状态为 `3` 或 `0` 的邮箱不计入主指标。

## 公司级核心业务指标

| 指标 | 公司数 | 获客成本 |
|---|---:|---:|
| 拥有至少一种有效联系方式的公司 | 250 | ¥1.45/家 |
| 拥有有效邮箱的公司 | 188 | ¥1.92/家 |
| 拥有有效电话的公司 | 196 | ¥1.84/家 |

三个指标都以公司为单位：同一家公司无论有多少个有效邮箱或电话，都只计算一次。三项成本均以累计 API 费用 ¥361.60 为分子，分别除以对应的公司数，不能相加。

## 邮箱技术审计

- `188 家公司`：188 家公司各自至少有一个状态为 `1` 的邮箱。这是公司级业务指标，`¥361.60 ÷ 188 = ¥1.92/家`。
- `277 个去重邮箱`：把全部有效邮箱文本放在一起跨公司去重后，共有 277 个独立地址；这是技术审计数量，不计算公司级获客成本。
- `284 条公司—邮箱关联`：同一个邮箱出现在两家公司行中会计算两次，因此是 284 条关联记录，不等于 284 个独立邮箱。该数字仅用于数据审计，不作为核心业务指标。

## 下一版流程

1. 搜索结果入库后，先按标准化公司名、官网域名、国家做法人实体去重；同实体直接复用已验证联系方式。
2. 公司联系方式只购买一次；原始响应以公司ID做幂等缓存。
3. API、官网、政府文件和人物邮箱进入同一增量验证队列，不再按来源分开漏验。
4. 无有效邮箱时先免费查官网；仍没有再逐公司搜索人物。只购买公司精确匹配且岗位相关的第一位人员。
5. 状态 `3` 多为验证服务器被反垃圾系统拦截，状态 `0` 表示接口已调用但没有完成检测；二者都不重复盲测，改走新人物或真实投递闭环。
6. 正式群发前先小批量发送，记录硬退信、软退信与回复；硬退信地址永久进入抑制名单。验证状态 `1` 是筛选门槛，不是送达保证。

本轮从 35 家扩充到 50 家时，没有再次调用高成本人物搜索：3 次增量公司搜索花费 ¥4.50，19 家公司联系方式花费 ¥19.00，新增 15 家有效联系方式公司，边际成本为 ¥1.57/家。该结果支持“先批量公司联系方式、状态1即停止、人物搜索仅作最后补救”的优化流程。

四个新品类从 0 扩充到 150 家时，搜索花费 ¥28.50、公司联系方式花费 ¥167.00，合计 ¥195.50，边际成本 ¥1.30/家。采用同义词搜索、严格本地过滤、公司ID全局去重和小批量停止策略后，没有使用人物搜索或重复验证状态0/3邮箱。

塑料桶从 0 扩充到 50 家时，`plastic bucket` 与 `plastic pail` 共搜索 6 页，花费 ¥9.00；购买 52 家公司联系方式，花费 ¥52.00；合计 ¥61.00，边际成本 ¥1.22/家。最后按剩余缺口小批量停止，未购买候选池中的其余联系方式。
