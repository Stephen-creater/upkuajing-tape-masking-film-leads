人物补救阶段先人工核对公司和职位，只购买6人的联系方式，实际恢复了 Samsung、Haier、Super Sales 三家公司的有效邮箱。人物搜索共花费 ¥19.50，是本轮最需要优化的环节：名称宽泛的公司容易返回大量错误实体。

## 下一版流程

1. 搜索结果入库后，先按标准化公司名、官网域名、国家做法人实体去重；同实体直接复用已验证联系方式。
2. 公司联系方式只购买一次；原始响应以公司ID做幂等缓存。
3. API、官网、政府文件和人物邮箱进入同一增量验证队列，不再按来源分开漏验。
4. 无有效邮箱时先免费查官网；仍没有再逐公司搜索人物。只购买公司精确匹配且岗位相关的第一位人员。
5. 状态 `3` 多为验证服务器被反垃圾系统拦截，状态 `0` 表示接口已调用但没有完成检测；二者都不重复盲测，改走新人物或真实投递闭环。
6. 正式群发前先小批量发送，记录硬退信、软退信与回复；硬退信地址永久进入抑制名单。验证状态 `1` 是筛选门槛，不是送达保证。

按本批数据估算，若前置实体去重、取消一次混合人物搜索，并避免对明显错误实体逐公司搜索，可至少节省约 ¥8.50；有效邮箱结果不应因此减少。
